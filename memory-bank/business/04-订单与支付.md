# 订单与支付模块

## 职责

订单创建、模拟支付、取消订单、状态管理、RabbitMQ 异步扣库存、WebSocket 实时推送。

## 订单状态流转

```
0=待付款 → 1=已付款 → 2=已完成
                 ↓
              3=已取消（只能从待付款取消）
```

## 核心流程

### 下单（createFromCart）
```
用户点"确认下单"
  → 查购物车 → 校验每本书库存 → 生成 OrderInfo（状态=0）
  → 生成 OrderItem 明细
  → 清空购物车
  → WebSocket 推管理员：/topic/admin/orders "新订单"
```
**注意：下单时只校验库存，不扣库存——支付时才扣。**

### 支付（pay）
```
用户点"去支付" → PaymentController.pay() → OrderServiceImpl.pay()
  → 校验：订单存在 + 本人 + 状态=0
  → 改状态=1（已付款）
  → 发 RabbitMQ 消息到 order.exchange (routing key: order.pay)
  → WebSocket 推用户：/topic/orders/{userId} "已付款"
  → WebSocket 推管理员：/topic/admin/orders "状态变更"

MQ 消费者 OrderPayConsumer.handlePay(orderId)
  → 查 OrderItem 明细
  → 逐条 call bookService.deductStock(bookId, quantity)
    → 乐观锁：selectById → setBooksNum → updateById → 版本号冲突重试3次
```

### 取消订单（cancel）
```
PUT /orders/{id}/cancel
  → 校验：本人 + 状态=0
  → 改状态=3
  → WebSocket 推用户 + 管理员
```

### 管理员改状态（updateStatus）
```
管理员后台下拉框改状态
  → 改状态
  → WebSocket 推对应用户 + 管理员
```

## 核心类
- `OrderController` — /orders/**
- `PaymentController` — /payment/**
- `OrderServiceImpl` — 下单/支付/取消/状态管理/统计/WebSocket推送
- `OrderPayConsumer` — RabbitMQ 消费者，异步扣库存
- `BookServiceImpl.deductStock()` — 乐观锁扣库存（@Version + 重试3次）
- `RabbitMQConfig` — 交换机/队列/绑定
- `WebSocketConfig` — STOMP 端点 + 消息代理

## WebSocket 推送频道

| 频道 | 谁订阅 | 收到什么 |
|------|--------|---------|
| /topic/orders/{userId} | 用户（OrderDetail.vue） | "已付款"、"已取消"、状态变更 |
| /topic/admin/orders | 管理员（OrderManage.vue） | "新订单"、"状态变更" |

## 注意事项
- 乐观锁需要数据库 book 表有 version 列 + MybatisPlusConfig 注册 OptimisticLockerInnerInterceptor
- RabbitMQ 用的是 CloudAMQP，virtual-host = 用户名
- 支付是模拟的——没有真实对接支付宝/微信支付，只是改状态
- 消费者扣库存失败只记日志不抛异常（Demo 级别），生产环境应加入死信队列
