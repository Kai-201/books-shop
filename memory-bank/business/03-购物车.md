# 购物车模块

## 职责

用户购物车——添加商品、修改数量、移除、清空。下单时校验库存。

## 核心流程

### 加入购物车
```
POST /cart → CartController.add()
  → 查 book 表确认库存
  → 查 cart 表看是否已存在（user_id + book_id）
    → 已存在 → quantity += 新增数量
    → 不存在 → INSERT
  → 联合唯一索引 uk_user_book 保证同一用户不能重复加同一本书
```

### 修改数量 / 移除 / 清空
```
PUT /cart/{id} → 修改 quantity
DELETE /cart/{id} → 移除单条
DELETE /cart → 清空当前用户购物车（下单时自动触发）
```

### 下单时的库存校验
```
createFromCart()
  → 遍历购物车每项 → selectById(bookId)
    → booksNum < quantity → 抛 BusinessException("库存不足")
    → 只校验不扣库存（支付时才扣）
```

## 核心类
- `CartController` — /cart/**
- `CartServiceImpl` — CRUD + 库存校验 + 自动合并
- `CartMapper` — MyBatis-Plus BaseMapper

## 注意事项
- 购物车有联合唯一索引 `uk_user_book(user_id, book_id)`——重复加入自动转 UPDATE 数量
- 下单成功后调 `cartService.clear(userId)` 清空购物车
- 购物车不涉及缓存，数据量小，直接查 MySQL
