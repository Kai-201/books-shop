# 认证与用户模块

## 职责

用户/管理员登录注册、JWT 签发与校验、Spring Security 权限控制、用户 CRUD 管理。

## 核心流程

### 登录流程
```
POST /auth/user/login
  → AuthServiceImpl.userLogin()
    → userMapper 查用户名
    → passwordEncoder.matches() 验密码
    → jwtUtil.generateToken() 生成 JWT（claims: id/loginName/role）
    → 返回 token + userInfo
```

### JWT 请求验证
```
前端请求（Header: Authorization: Bearer xxx）
  → JwtAuthenticationFilter.doFilterInternal()
    → 取 token → jwtUtil.parseToken()
    → 得到 LoginUser(id, loginName, role)
    → role 映射为 ROLE_admin / ROLE_user
    → 放入 SecurityContextHolder
```

### 权限校验两层
1. URL 级：`SecurityConfig.authorizeRequests()` — /users/** 需 admin
2. 方法级：`@PreAuthorize("hasRole('admin')")` — Controller 方法上加

### 注册
```
POST /auth/user/register
  → 检查 loginName 重复
  → passwordEncoder.encode() 加密
  → 插入 user_info 表
```

## 核心类
- `AuthController` — /auth/**
- `AuthServiceImpl` — 登录/注册逻辑
- `UserController` — /users/**（需要 admin 角色）
- `UserServiceImpl` — 用户 CRUD
- `JwtUtil` — Token 生成/解析（HS256, secret=yaml 配置, 24h 过期）
- `JwtAuthenticationFilter` — 拦截器，从 Header 提取 Token
- `SecurityUtils` — 静态工具类，获取当前登录用户
- `SecurityConfig` — Spring Security 过滤器链配置

## 注意事项
- 前端 localStorage 分 `userToken` 和 `adminToken`，互斥
- 401/403 返回 JSON（不是默认的重定向），由 GlobalExceptionHandler + SecurityConfig 的 entryPoint 处理
- 密码用 BCrypt 加密，不是 MD5
- 角色只有两种：admin/user，不是完整的 RBAC 三模型
