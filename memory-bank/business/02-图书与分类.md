# 图书与分类模块

## 职责

图书 CRUD、分类管理、ES 全文搜索、Redis 三级缓存。

## 核心流程

### 图书查询（带三级缓存）
```
getById(id)
  → BloomFilter.mightContain(id) → false → 直接返回 null（防穿透）
  → Redis 缓存 book::{id}
    → 命中 → 返回
    → 未命中 → @Cacheable(sync=true) → 单线程查 DB → 返回 + 写缓存（防击穿）
```

### 缓存 TTL（防雪崩）
- books 组（列表/分页）：25~35 分钟随机
- book 组（单本详情）：25~35 分钟随机
- categories 组：50~70 分钟随机
- 默认：8~12 分钟随机

### 图书写操作 → 清缓存 + 同步 ES
```
create() / update() / delete()
  → @CacheEvict(cacheNames={"books","book"}, allEntries=true) 清缓存
  → bookSearchService.sync() / remove() 同步 ES
  → create 时 bloomFilter.put(id) 更新布隆
```

### ES 搜索 vs MySQL 分页
```
前端搜索框有 keyword：  GET /api/books/search?keyword=Java → ES multi_match(bookName, bookAuthor, bookSn)
前端浏览（无 keyword）： GET /api/books/page?page=1&size=100 → MySQL 分页（走缓存）
```

### ES 数据同步
```
首次启动：@PostConstruct → 创建索引 books → 全量 SELECT MySQL → 逐条 PUT ES
运行时：BookServiceImpl.create()/update() 末尾调 sync()，delete() 末尾调 remove()
```

## 核心类
- `BookController` — /books/**
- `BookServiceImpl` — 缓存注解 + 布隆 + ES 同步
- `BookSearchService` — ES 的 sync/search/remove + 启动初始化
- `BookDocument` — ES 索引文档结构（DTO）
- `BookCategoryController` / `BookCategoryServiceImpl`
- `RedisCacheConfig` — JSON 序列化 + 随机 TTL
- `ElasticsearchConfig` — RestTemplate Bean（Basic Auth + SSL）
- `SslUtils` — 信任所有证书的 RequestFactory

## 注意事项
- ES 9.x 客户端和 Spring Boot 2.7 的 HttpClient 版本冲突，用的是 RestTemplate 直接调 HTTP API
- BookDocument 与 BookVO 是分离的——Document 给 ES 搜，VO 给前端看
- uploaderId=NULL 表示管理员上架，toVO 里拼 uploaderType="admin" 和 uploaderName="管理员"
- toVO 有 N+1 问题——通过 loadUploaders() 批量查用户 + HashMap 解决
- 布隆在内存里，重启后从 MySQL 重建
